## v1.2.0 (2026-03-15)

### 🐛 버그 수정

#### 프로젝트 목록 페이지
- **하위/최하위 프로젝트 레벨 뱃지 오류 수정**
  - 기존: 렌더링 트리의 깊이(depth) 기준으로 레벨 계산 → 잘못된 뱃지 표시
  - 수정: 실제 DB 부모 체인을 탐색하는 `getProjectLevel()` 함수 기준으로 레벨 계산
  - API 응답에 `parent: { id, parentId }` 데이터 포함 추가

#### 담당자별 업무 페이지
- **하위 프로젝트가 목록에 표시되지 않는 오류 수정**
  - 기존: 트리 구성 시 최하위(depth=3) 프로젝트의 상위 노드를 잘못된 depth로 등록 → 하위(depth=2) 매칭 실패
  - 수정: depth별 독립 맵(`upper1Map`, `lower2Map`, `lower3Map`) 사용으로 트리 구조 정확하게 재구성
- **업무 체크/해제 시 선택된 사용자가 초기화되는 오류 수정**
  - 기존: 체크 토글 후 `fetchUsers()` 호출 시 항상 첫 번째 사용자로 리셋
  - 수정: `preserveSelection` 파라미터 추가, 체크 토글 시 현재 선택 사용자 유지

#### 프로젝트 폼 (하위/최하위 프로젝트 생성·수정)
- **최하위 프로젝트 담당자 지정이 저장되지 않는 오류 수정**
  - 기존: 부모 프로젝트 로딩 전에 폼이 렌더링되어 `projectLevel=0`으로 잘못 계산 → 담당자 없이 저장
  - 수정: `parentLoading` 상태 추가, 부모 프로젝트 fetch 완료 후에 폼 렌더링
- **할 일 입력 후 + 버튼/Enter 없이 저장해도 반영되지 않는 오류 수정**
  - 기존: 입력 중인 텍스트(`newTaskInputs`)가 저장 요청에 포함되지 않음
  - 수정: 제출 시 미확정 입력값을 `finalAssignees`에 병합 후 저장

### ✨ 신규 기능

#### 담당자별 업무 페이지 트리 구조 개편
- 최상위 프로젝트는 섹션 헤더로만 표시
- 상위 프로젝트 → 하위 프로젝트 → 최하위 프로젝트 순으로 계층적 표시
  ```
  최상위 프로젝트 (헤더)
  ├─ 상위 프로젝트
  │   ├─ 하위 프로젝트
  │   │   └─ 최하위 프로젝트
  ```

#### 프로젝트 폼 할 일 입력 UI 추가
- 하위/최하위 프로젝트에서 담당자 지정 시 할 일 항목 직접 추가 가능
- `+` 버튼, Enter 키, 또는 저장 버튼으로 할 일 등록
